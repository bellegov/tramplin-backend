Платформа «Трамплин» — Backend API
«Трамплин» — это интерактивная экосистема для централизованного взаимодействия студентов, работодателей и карьерных центров вузов. Платформа позволяет студентам находить стажировки и вакансии, компаниям — верифицировать свой статус и подбирать таланты, а кураторам — эффективно модерировать контент.
✨ Ключевой функционал
🎓 Для Соискателей (Студентов)
Интерактивная карта: Поиск вакансий и ивентов по геолокации.
Личный кабинет: Управление резюме, история откликов со статусами.
Нетворкинг: Система друзей, поиск коллег и просмотр их «карьерных интересов».
Рекомендации: Возможность рекомендовать друга на конкретную вакансию (социальный лифт).
Система уведомлений: «Колокольчик» с оповещениями о новых вакансиях от подписок.
🏢 Для Работодателей (Компаний)
Верификация по ИНН: Двухэтапная проверка куратором для защиты от фейков.
Управление вакансиями: Полный CRUD с поддержкой архивации и мягкого удаления (Soft Delete).
Работа с откликами: Удобный трекинг кандидатов (Принят / Отклонен / Резерв) с просмотром их профилей и рекомендаций.
🛡️ Для Кураторов (Администраторов)
Панель модерации: Одобрение заявок компаний, управление всеми пользователями.
Контроль контента: Возможность принудительного редактирования профилей и удаления медиафайлов.
🛠 Стек технологий
Core: Java 17, Spring Boot 3.2 (Web, Data JPA, Security).
Auth: Stateless JWT (JSON Web Token) с ролевой моделью доступа.
Database: PostgreSQL 15 (сложные реляционные связи, индексация).
Media Storage: MinIO (S3-compatible) — облачное хранилище для аватаров и обложек.
DevOps: Docker & Docker Compose (полная контейнеризация всей инфраструктуры).
API: Swagger UI (OpenAPI 3.0) для интерактивной документации.
🏗 Архитектурные решения (Аргументация)
Модульный монолит: Выбран для высокой скорости разработки и обеспечения целостности данных на этапе конкурса, с возможностью легкого распила на микросервисы.
Soft Delete: Вакансии не удаляются физически, что сохраняет целостность истории откликов в личных кабинетах студентов.
S3 Storage: Использование MinIO вместо хранения картинок в БД или файловой системе сервера повышает производительность и масштабируемость.
CORS & Security: Бэкенд полностью настроен для работы с современными фронтенд-фреймворками (React/Vue/Angular).
🚀 Быстрый запуск
Для запуска всей инфраструктуры (API, БД, Хранилище) вам понадобится только Docker.
Клонируйте репозиторий.
Выполните команду в корневой папке:
code
Bash
docker-compose up --build -d
Готово! Сервисы доступны по адресам:
Backend API: http://localhost:8080
Swagger UI: http://localhost:8080/swagger-ui/index.html
Adminer (БД): http://localhost:8081
MinIO Console: http://localhost:9001 (login: root, pass: rootpassword)

📂 Структура проекта
config/: Конфигурации безопасности, CORS, OpenAPI и инициализация данных.
controller/: REST-контроллеры, разделенные по зонам ответственности.
dto/: Объекты передачи данных (Records) для чистого API.
model/: Сущности БД (JPA Entities) и перечисления (Enums).
repository/: Интерфейсы доступа к данным Spring Data JPA.
service/: Бизнес-логика приложения.
security/: Логика работы JWT фильтров и генерации токенов.
